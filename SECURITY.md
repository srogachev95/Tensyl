# Security Policy

## Supported Versions

Tensyl is pre-1.0 software. Security fixes are applied to the latest released
minor version when a public release exists. Older pre-1.0 versions may receive a
fix when the change is low risk, but users should expect to upgrade to the
latest release.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately through GitHub Security
Advisories for this repository. If advisories are not available, open a minimal
issue asking for a private security contact without including exploit details.

For public issues, keep examples limited to impact and affected versions until a
fix is available. Mechanics bugs can be boring; security bugs should be boring
only after they are fixed.
